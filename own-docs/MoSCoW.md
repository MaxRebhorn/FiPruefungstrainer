M - _Must have_, S - _Should have_, C - _Could have_, W - _Won't have_.


# Must have

  - SQL aufgaben Simulieren
  - GET aufgaben
  - CREATE Aufgaben
  - UPDATE aufgaben
  - GRANT aufgaben
  - Für jeden Aufgabenstatz soll eine DB erstellt  oder angepasst werden können
  - Pseudocode Aufgaben Simulieren
  - Anstatt Pseudocode sollte hier dann Javascript verwendet werden
  - IHK aufgaben sollten hierzu verwendet werden mit Mocks
  - Die Notwendigen Mocks sollten weg abstrahiert sein so dass der Nutzer sie nicht bemerkt


# Should have

  - Leichtes einfügen von neuen alten IHK aufgaben
  - Tracking von Lernprozess
  - Aufgaben nach Aufgabenstellungen gruppieren

# Could have

  - Auswahl von verschiedenen Programmiersprachen für Pseudocode aufgaben
  - Support für ERM, SEQUENZ, UML  Diagramm aufgaben mit visuellem design tool
  - Support für Text aufgaben, via einbindung von API key um antworten mit fragestellung an KI zu schicken die dann die aufgabe auswertet
  - Ranking basierend auf schwierigkeit
  - Timer für Aufgaben
  - Angabe von anzahl an versuchen , hardcore modus mit nur einem test versuch pro aufgabe
# Won't have

  - Automatiesiertes umwanlden von IHK prüfung pdfs in aufgaben
